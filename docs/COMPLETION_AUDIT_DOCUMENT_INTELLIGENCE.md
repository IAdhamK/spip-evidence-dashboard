# Completion Audit Document Intelligence V2

Tanggal audit: 13 Juli 2026
Provider runtime pengembangan: Sumopod `deepseek-v4-pro`
API surface: Responses API dengan fallback Chat Completions yang tervalidasi

## Kesimpulan

Rangkaian engine V2, hard gate, workflow review, serta fondasi rollout telah diimplementasikan dan dapat dijalankan tanpa mengganti atau menghapus pipeline legacy. Scope yang dapat dibuktikan secara lokal lulus pengujian. Korpus operasional nyata sudah tersedia dan telah dipakai untuk failure-mode analysis, tetapi belum direview sebagai pilot domain atau expert gold. V2 belum boleh disebut general-release complete karena rule domain belum disahkan, holdout Evaluasi, korpus Learning, dan pilot bertanda tangan belum selesai, dan beberapa fidelity parser/operational capability masih perlu dituntaskan.

Real upload tetap nonaktif dan `ALLOW_PARTIAL_PRIMARY=false`. Import legacy, hasil coverage parsial, rule draft, kutipan yang tidak cocok dengan unit sumber, periode/organisasi ambigu, overgrade, verifier reject, dan run tanpa human approval semuanya fail-closed.

Pemulihan Visual/OCR tidak lagi menunggu satu dokumen selesai. Kebijakan `unit-checkpoint-v2` menyimpan manifest seluruh unit dan hasil sukses setiap batch, memverifikasi key/tipe/urutan/heading/status/source/text/warning/metadata dengan checksum kanonis saat resume, memakai ulang hanya snapshot lengkap yang valid, dan mengulang unit yang belum sukses. Checkpoint stale atau tampered ditolak; kegagalan menulis checkpoint menggagalkan run. Lease recovery menyimpan superseded-run event dan pointer resume secara atomik sebelum run baru dibuat. Regression PDF dua halaman mematikan proses dua kali, mempertahankan payload/lineage, mencapai claim ketiga, dan membuktikan hanya halaman kedua diproses ulang. Regression shutdown PDF empat halaman memasang `stopping` sebelum join, menolak enqueue konkuren, mempertahankan leader selama drain, menolak manager kedua, requeue payload secara aman, lalu memproses hanya halaman 3–4. Attempt fencing menolak renew/attach/supersede/requeue/complete/fail dari claim lama. Probe liveness tetap 200 dan readiness menjadi 503 selama stopping/draining tanpa mengekspos alasan internal. Bukti lain tersedia melalui stage checkpoint, audit event content-free, config API, metric job-recovery/draining beserta durasi drain, alert active recovery loop, serta alarm kritis drain macet setelah sepuluh menit.

## Gate yang harus dipenuhi sebelum tahap rollout

1. Domain owner mengesahkan checksum seluruh rule yang akan aktif.
2. Minimal 50 expert-labelled cases memenuhi Recall@5 ≥95%, source accuracy ≥95%, overgrade ≤2%, dan regression threshold yang disepakati.
3. Reviewer terautentikasi menjalankan pilot dokumen nyata; product owner menandatangani comparison report.
4. OCR lokal dapat dipakai setelah runtime/self-test dan fidelity locator tervalidasi; vision eksternal tetap memerlukan capability approval dan consent terpisah.
5. Deployment menyiapkan alert sink/dashboard; gunakan satu replica selama queue masih berbasis SQLite.
6. Database dan payload root berada pada volume terenkripsi yang dibuktikan platform; platform owner menerbitkan attestation bertanda tangan dari namespace runtime, memasang key/evidence private, kemudian mengaktifkan `ANALYSIS_PAYLOAD_STORAGE_ENCRYPTION_VALIDATED=true`. Gate canary tetap tertutup bila flag, signature, expiry, owner/permission, atau path/device binding tidak lulus.
7. Canary dan general release mengikuti dua siklus stabil sebelum legacy V1 didepresiasi.

Dokumen ini adalah audit bukti implementasi, bukan pengganti sign-off domain atau keputusan produksi.
