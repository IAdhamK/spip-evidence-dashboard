## Aturan pemeliharaan

1. Jangan menutup item hanya karena mitigasi sementara tersedia; lampirkan bukti exit condition.
2. Perubahan severity atau owner role harus mempunyai alasan dan audit date.
3. Item human/external tidak boleh diisi dengan approval, label, atau release evidence sintetis.
4. Tambahkan failure baru dari pilot/incident ke register sebelum rollout dilanjutkan.
5. Release ledger dan dokumen ini harus konsisten; dua stable cycle tidak boleh dicatat jika TD-001, TD-002, TD-006, TD-007, atau TD-010 masih terbuka.
